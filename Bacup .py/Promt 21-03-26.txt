PROMPT DI ARCHITETTURA TECNICA: SISTEMA "MYPLAYR"
Ruolo: Agisci come Sviluppatore Full-Stack esperto in Python, Automazione Video e Cloud Database.

1. PANORAMICA DEL SISTEMA
Il sistema MyPlayr è una piattaforma di video-analisi sportiva composta da due software separati che comunicano tramite Supabase:
App Web (Frontend/Admin): Gestisce l'interfaccia utenti, le iscrizioni, la visualizzazione dei video e l'invio dei comandi di taglio.
Regista (Backend Locale/Mini PC): Un software che gira sul campo, legge i comandi dal Cloud, controlla le telecamere e usa FFmpeg per processare i video Master in 4K.

2. STACK TECNOLOGICO E STRUTTURA CODICE
Linguaggio: Python 3.x.
Interfaccia: Streamlit (Layout wide, tema Dark personalizzato #2F353B, accenti verdi #28a745).
Database Cloud: Supabase (PostgreSQL) con tabelle:
utenti: Profili atleti e admin (ruolo, email, nickname, bio, ig_tag).
calendario: Programmazione match (stato: PROGRAMMATO / FATTO).
comandi_clip: Coda di lavoro per il Mini PC (id_partita, inizio, durata, stato: RICHIESTO / COMPLETATO).
clip_generate: Archivio degli highlight (url_video, likes, pubblico [bool]).
Processamento Video: FFmpeg integrato tramite la libreria subprocess. Utilizzo del comando -c copy per tagli ultra-rapidi senza perdita di qualità.
Storage: Sincronizzazione tramite Google Drive Desktop (Disco G:) e cartelle locali (ARCHIVIO_PARTITE, CLIP_TAGLIATE).

3. LOGICA DEI FILE SORGENTE
File app.py (Il "Cervello" Web)
Navigazione Dinamica: Gestita tramite st.session_state.pagina con struttura a catena if / elif / elif per evitare sovrapposizioni di contenuti.
Protezione: Password sviluppatore iniziale e sistema di Login che verifica le credenziali e il ruolo (admin o Player) su Supabase.
Pagina Admin: Dashboard con statistiche (query count su Supabase) e form per inserire nuovi match nella tabella calendario.
Pagina Partite: Player video per i match con stato FATTO e interfaccia per richiedere il taglio di una clip (inserimento riga in comandi_clip).
Hall of Fame: Galleria dei video pubblici ordinati per numero di likes.
Logica "Regista" (Automazione Mini PC)
Il codice monitora costantemente la tabella calendario. Quando l'ora attuale coincide con un match PROGRAMMATO, avvia la registrazione locale.
Monitora la tabella comandi_clip: quando trova una richiesta con stato RICHIESTO, lancia FFmpeg sul file Master nel disco G:, genera la clip e carica l'URL nel database, aggiornando lo stato in COMPLETATO.

4. FUNZIONI CRITICHE IMPLEMENTATE
Integrazione Email: smtplib con MIMEText per invio newsletter e conferme iscrizione.
Sicurezza Percorsi: Utilizzo di os.path.join e BASE_DIR per rendere il software portabile tra Windows e Linux (Streamlit Cloud).
Gestione Errori: Blocchi try/except su ogni chiamata database e controllo esistenza file fisici prima del caricamento video.
